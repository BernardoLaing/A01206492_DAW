Definiciones y preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red de muchas redes, y la World Wide Web es un modelo para compartir información por medio del Internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

HTTP es Hyper Text Transfer Protocol, y provee métodos de comunicación entre clientes y servidores.

  • GET: pide la representación de un recurso. Sólo se debe de usar para recuperar datos.
  • HEAD: es similar a GET, pero pide una respuesta sin cuerpo.
  • POST: Pide al servidor aceptar una entidad.
  • PUT: Pide al servidor cambiar la representación de un recurso con el cuerpo de la solicitud.
  • PATCH: Se utiliza para envíar modificaciones parciales a un recurso.
  • DELETE: Elimina el recurso especificado.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Los códigos HTTP son respuestas de parte del servidor al cliente. El código 200 significa OK, por lo que no hubo ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Los códigos 400 son errores de parte del cliente, y el 404 significa que el recurso que se pide no está en el servidor. No es responsabilidad del desarrollador arreglarlo, ya que es el usuario buscando algo que no existe.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Los códigos 500 son errores de parte del servidor. El error 500 es un mensaje de error genérico, y es algo que debería de ser corregido por el desarrolador.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que algún elemento de HTML, que se puede seguir usando, ha sido reemplazado por una mejor alternativa, ya sea en html o css. Algunos elementos desaprobados en HTML5 son:

  • <center>
  • <dir>
  • <font>

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML 4 y HTML 5 tienen etiquetas diferentes. HTML 5 cuenta con meta tags, <meta>. En HTML se pueden incluir elementos de SVG y MathML.

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla, <table>, tiene un encabezado definido como <thead>, un cuerpo <tbody>, filas <tr>, y celdas <td>. También se pueden incluir encabezados de columna, <th>, y footers <tfoot>

Ciclo de vida de un sistema de información

El ciclo de vida de un sistema es todo el tiempo desde que se tiene la idea, hasta que "muere" el sistema, o es desechado. Comienza con estudios de la idea, análisis de la viabilidad del proyecto, estudios de mercado, sigue con todo el proceso de ingeniería de requerimientos y diseño del sistema, hasta la implementación, pruebas, y todo el tiempo que el sistema esté en operación.

Ciclo de desarrollo de sistemas de información

El ciclo de desarrollo es el proceso de ingeniería que se lleva para el desarrollo de software. Este ciclo es una estructura con una secuencia de pasos para realizar el proyecto. De manera general, estos pasos son:

  • Comunicación
  • Requisitos del Sistema
  • Estudio de Factibilidad
  • Análisis del Sistema
  • Diseño de Software
  • Códigos

Lab 3: CSS

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

!important puede ser muy util, pero normalmente no es considerado una buena práctica, ya que hace el código más dificil de leer y mantener. Puede llegar a ser necesario para hacer override a alguna regla de un archivo css de terceros, cómo importando algún elemento o framework.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Se tiene que tener cuidado con el tamaño y resolución de la imagen, ya que si no se ajusta correctamente, puede no verse bien en ciertos dispositivos.

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Recomendaría usar %, px, o de preferencia, em. El problema con usar pt, y otras medidas absolutas para la fuente, como cm o in, es que existen diferentes tamaños de pantalla, y cosas que se podrían ver en una computadora de escritorio, podrían ser demasiado grandes para algo como una pantalla de celular. En pocas palabras, usar algo como em o %, aumenta la resposividad de la página.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Las versiones minimizadas eliminan todo lo inecesario del archivo css, siendo un archivo que por el navegador se lee de la misma manera, pero ocupando menos espacio, por lo que la carga de los archivos, y por lo tanto, de la página, es más rápida.
