Preguntas a responder

¿Por qué es una buena práctica usar JavaScript checar que sean válidos los inputs de las formas antes de envíar los datos al servidor?

Validando los inputs se puede comprobar si el usuario te está dando las entradas necesarias, por ejemplo, cumpliendo con todos los requisitos de una contraseña, e informarle si algo está mal, o si falta de llenar algún campo requerido.

¿Cómo puedes saltarte la seguridad de las validaciones hechas con JavaScript?

Entrando a ver las herramientas de desarrollador, desactivando el uso de JavaScript en la configuración del navegador.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Porque de esta manera, no se necesita envíar nada al servidor para validar, y así no se tiene que recargar la página para volver a ingresar los datos, haciéndolo más conveniente para el usuario en caso de equivocarse en la forma. De cualquier manera, siempre se deben de hacer validaciones del lado del servidor.
